Project Goals

To produce an intuitive application to allow users to access information surrounding photo editing

To provide advice on a range of photo editing topics

To allow users to practice making a range of edits without having to leave the tutorial

To provide a mobile-first application that incorporates a range of modern features

Project Deliverables

Based on the above goals, I then identified a list of features that my application must have in order to successfully meet these key aims:

Application Requirements

  • Provide an intuitive user interface
  • Produce a Swift application that's compatible with the current range of iOS devices
  • Base my user interface on the SwiftUI framework
  • Ensure my user interface integrates well with the OS

Essential Image Processing Features

  • Exposure
  • Contrast
  • Cropping
  • Greyscale
  • Brightness

Advanced Image Processing Features

  • Noise Reduction
  • Spot Healing
  • Graduated Filter
  • Saturation
  • Haze Reduction

Planning

As part of the planning process, I:

  • Produced a formal project plan
  • Undertook a literature survey to research the state-of-the-art for various techniques
  • Produced a project schedule in the form of a Gantt chart
  • Produced initial drafts of the algorithms
  • Produced a series of iterative user interface drafts:

This draft introduces the gamification elements in the application, as well featuring a heavily text-based home screen and tutorial screen featuring multiple information panes.

The homescreen was changed to a more graphics-based layout, the tutorial interface divided into separate content and practice areas, and user profiling capability added.

The tutorial area now features interactive elements integrated with the teaching content, as well as a redesigned UI that better adheres to the iOS design language.

Development

As I'd never coded in Swift or CIKernel before, I first began by learning the basics of both languages and identifying which areas I'd need to be proficient in to complete the project successfully. Once I was comfortable in using both, I began to build my implementation and gradually add the features specified in the plan.

During development, I met with my supervisor each week to review progress and identify what I should do next. These meetings also allowed me to get opinions on new ideas I had for the project during development, such as adding cloud synchronisation. I completed development as per my development schedule, with all of the pre-determined abilities present as well as some new ones added along the way.
